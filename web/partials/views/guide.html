Getting Started

1. Create Your First Job

Go to Jobs and click + Create Job. Pick a task type:

  • Shell — run a command: echo "Hello from Kronforce"
  • HTTP — call an API: GET https://httpbin.org/get

Set a schedule (cron, one-shot, or on-demand) and click Save.

2. Trigger and Watch

Click the ▶ Trigger button on your job. Go to Executions to see the result — output, exit code, duration, and timestamps.

3. Set Up Notifications

Get alerted when jobs fail. Go to Settings → Notifications and configure at least one channel:

  • Email — SMTP server (Gmail, SendGrid, etc.)
  • Slack — paste an incoming webhook URL
  • Teams — paste a Teams webhook URL
  • PagerDuty — paste your Events API URL with routing key
  • SMS — Twilio or any SMS webhook

Then enable On failure on any job's Alerts tab.

4. Organize with Groups

Create groups like ETL, Monitoring, Deploys to organize jobs. Go to Groups and click + New Group.

5. Chain Jobs Together

Jobs can depend on other jobs or react to events:

  • Dependencies — on the Schedule tab, add jobs that must succeed first
  • Event triggers — schedule type "Event" fires when another job completes, fails, or outputs a pattern
  • Output extraction — extract values from stdout and write them to global variables for the next job to use with {{VAR_NAME}}

Visualize the dependency graph on the Map page.

6. Add a Remote Agent

Run jobs on other machines by starting an agent:

KRONFORCE_CONTROLLER_URL=http://this-server:8080 \
KRONFORCE_AGENT_KEY=kf_your_agent_key \
KRONFORCE_AGENT_NAME=my-agent \
cargo run --bin kronforce-agent

Or use a custom agent in Python/Go/Node — see Docs → Custom Agents.

7. Secure Your Instance

  • API keys — create scoped keys for teams in Settings → API Keys
  • SSO — enable OIDC login (Okta, Azure AD, Google) with KRONFORCE_OIDC_ISSUER env var
  • Approval workflows — require approval on critical jobs (Advanced tab)
  • SLA deadlines — set completion deadlines with warnings (Advanced tab)

8. Monitor

  • Dashboard — execution timeline, charts, and stats at a glance
  • Prometheus — scrape /metrics for Grafana dashboards
  • Events — activity feed with severity filtering
  • Audit log — who changed what and when (Settings → API: GET /api/audit-log)

Need help? Check the page for detailed API reference, workflow patterns, and custom agent protocol. Or visit GitHub for issues and discussions.
